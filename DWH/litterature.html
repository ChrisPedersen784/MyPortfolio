Week 1

This week, I've learned about Data Warehousing (DWH) basics.

A Data Warehouse is a large data repository built on top of a database. It's different from a typical database and is filled with data through the ETL (Extract, Transform, Load) process.

I explored the difference between Online Transaction Processing (OLTP) and Online Analytical Processing (OLAP). OLTP focuses on day-to-day operations, while OLAP is designed for analytical queries.

Benefits of a Data Warehouse include easy data access, improved analysis, better security, and efficient handling of large datasets. I also looked into Data Lake vs. Data Warehouse, understanding their specific use cases.

Data Mart, a smaller version of a Data Warehouse, was introduced, catering to specific departmental needs. The Extract, Transform, Load (ETL) process, involving gathering, cleaning, and loading data, was explored, along with tools like SSIS and Informatica.

Traditional Data Warehousing solutions like Teradata, IBM Db2, Oracle Database, Microsoft SQL Server, and MySQL were briefly covered, each with its own features and industry use cases.

In summary, this week's learning revolved around the essentials of Data Warehousing, covering OLTP vs. OLAP, Data Lake vs. Data Warehouse, Data Mart, ETL processes, and traditional solutions.
